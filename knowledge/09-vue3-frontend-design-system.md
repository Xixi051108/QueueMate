# Vue3 前端与设计系统实现记录

## 本轮目标

在后端接口保持不变的前提下，为空的 `frontend/queuemate-web` 初始化 Vue3 应用，并先完成普通用户最短业务闭环。设计先于页面实现，统一规则保存在 `design-system/MASTER.md`。

## 技术实现

- Vue 3 + Vite
- Vue Router 路由与角色守卫
- Element Plus 组件和图标
- Axios 请求、Bearer Token 注入与统一错误处理
- CSS 语义 tokens 和移动优先响应式布局
- 路由级动态导入与 Element Plus 组件按需注册

用户端页面包括：

```text
登录
  -> 地点列表
  -> 地点详情
  -> 选择时段并预约 / 现场取号
  -> 我的预约 / 钱包 / 我的排队
```

## 设计决策

QueueMate 采用“城市服务台”方向：服务蓝负责导航与主要操作，状态色只用于明确反馈，场所、预约、消费码和排队号码使用克制的票据式信息结构。页面不使用渐变、玻璃效果、装饰动画和大面积胶囊圆角。

设计系统统一定义：

- 颜色与对比度
- 中文字体栈与数据字体
- 4px 间距体系
- 4/8/12px 圆角
- 阴影和边框
- 按钮、表单、卡片、状态、票据和反馈规则
- 375/768/1024/1440px 响应式检查点
- 键盘焦点、44px 触控目标与减少动画设置

## 验证

生产构建：

```powershell
cd D:\QueueMate\frontend\queuemate-web
pnpm install
pnpm build
```

最后一次构建成功。登录、地点列表、地点详情、我的预约、钱包和我的排队页面使用 Chrome DevTools 设备模拟，在 375px 与 1440px 共 12 个页面/视口组合中完成检查：没有横向溢出，没有浏览器运行时错误。

视觉 QA 使用符合当前 DTO 的临时本地响应，不写入产品代码或数据库。本会话后端未启动，因此仍需在后续联调中使用真实 Spring Boot + MySQL 验证登录、预约、充值、取消和取号请求。
